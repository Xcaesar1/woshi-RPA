# 审查与提交规范

更新时间：2026-04-27

## 1. 目标

每次提交都要让后续接手的人知道：为什么改、改了什么、怎么验证、有没有沉淀长期规则。

## 2. 提交前检查

提交前至少确认：

- 没有提交账号、密码、Cookie、密钥、本地配置。
- 没有提交无关运行目录、缓存、大体积临时文件。
- 业务规则变化已经更新 `docs/business/`。
- 执行流程变化已经更新 `docs/harness/`。
- 项目运行方式变化已经更新 `docs/project/`。
- 已运行合适验证，或明确说明未验证原因。

## 3. 提交说明建议

提交信息不使用空泛描述，优先写清楚业务效果。

推荐：

```text
Route UPS shipments to packing output
Add Harness docs for RPA execution workflow
Fix Beijing local time display in task logs
```

不推荐：

```text
update
fix bug
v1.1
```

## 4. 审查重点

审查时优先看：

- 是否影响现有任务结果。
- 是否破坏 Docker、Redis、Worker、浏览器单槽边界。
- 是否改变 Excel 输出规则。
- 是否有回归风险。
- 是否有必要的文档或测试沉淀。
