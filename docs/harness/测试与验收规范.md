# 测试与验收规范

更新时间：2026-04-27

## 1. 验收原则

项目交付不能只看“代码没有报错”，而要看业务人员能不能直接使用结果。

默认验收顺序：

```text
能运行
  -> 能生成结果
  -> 结果内容正确
  -> 样式和命名正确
  -> 异常可解释
  -> 规则已沉淀
```

## 2. Web 任务系统验收

需要检查：

- 提交 `.txt` 清单成功。
- 提交 `.xlsx` 清单成功。
- 粘贴 FBA 成功。
- 非法输入不会入队。
- 列表页能看到状态。
- 详情页能看到阶段、计数和最近日志。
- 终态任务能下载结果。
- 单个业务结果时直接下载 `.xlsx`。
- 多个业务结果时下载 zip，里面只包含业务需要的 `.xlsx`。

## 3. 浏览器自动化验收

需要检查：

- 登录态有效时可直接进入系统。
- 登录态失效时可用本地配置自动登录。
- 能进入 `FBA -> FBA货件`。
- 能按 FBA 精确搜索。
- 能进入正确详情页。
- 能识别仓库卡片。
- 能点击下载按钮。
- 能选择导出不包含图片。
- 下载文件进入当前任务独立目录。
- 失败时保留截图、URL、标题和错误原因。

## 4. Excel 输出验收

第一种货代信息表检查：

- 文件名店铺简称、品线、日期正确。
- Sheet 名和票数正确。
- 标题、表头、明细、汇总行正确。
- 查表字段正确。
- 边框、字体、字号、合并单元格正确。

第二种 UPS 装箱信息表检查：

- 分流依据为源文件内部 `货件名称` 是否包含 `UPS`。
- 同一个 FBA 目录内 UPS 文件合并为一份工作簿。
- 不同 FBA 之间不合并。
- Sheet 名为 `共N票M件`。
- 仓库地址黄色底、红字、字号 12。
- 条码列黄色底、红字。
- 含中文字体用宋体，不含中文字体用 Arial。

## 5. 队列和并发验收

需要检查：

- 多人可以同时提交。
- Redis 队列计数正常。
- worker 串行领取任务。
- 浏览器执行槽同一时间只允许 1 个任务。
- worker 重启后能恢复队列。
- 心跳超时的 RUNNING 任务不会永久卡死。

## 6. Docker 验收

需要检查：

```powershell
docker compose ps
curl http://127.0.0.1:8000/healthz
curl http://127.0.0.1:8000/api/system/status
```

期望：

- `web` healthy。
- `worker` running。
- `redis` running。
- 系统状态接口能返回队列、运行任务、worker 和浏览器槽状态。

## 7. 交付前必须说明

最终交付时必须说明：

- 完成了什么。
- 验证了什么。
- 哪些没有验证。
- 是否有风险或限制。
