# 失败复盘模板

更新时间：2026-04-27

每次线上任务、真实站点执行、Excel 输出或部署出现问题后，优先用本模板沉淀。目标不是追责，而是减少同类问题再次发生。

## 1. 基本信息

```text

发现人：

相关 FBA：
影响范围：
当前状态：已修复 / 处理中 / 暂缓
```

## 2. 用户看到的现象

说明业务人员实际看到的问题：

- 页面报错是什么。
- 下载结果缺了什么。
- Excel 哪个字段、样式或文件名不对。
- 任务状态卡在哪里。

## 3. 复现方式


## 4. 根因分析

优先写清楚“为什么会发生”，不要只写“代码有 bug”。

常见根因类型：

- 业务规则理解错误。
- 页面 DOM 或按钮变化。
- Excel 源文件格式变化。
- 查表数据缺失或重复。
- 文件命名或目录隔离问题。
- Docker / Redis / SQLite / 浏览器环境问题。
- 时间、时区、编码、字体或样式问题。

本次问题至少选择一项沉淀：

- 更新业务规则文档。

- 更新操作规范。
- 更新页面提示。
