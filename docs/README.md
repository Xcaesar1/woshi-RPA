# 项目文档导航

更新时间：2026-04-27

本文档是领星自动化 RPA 项目的长期文档入口。根目录只保留代码、配置、样例和运行入口；项目说明、业务规则、Harness 流程统一放在 `docs/` 下。

## 文档分类

| 目录 | 用途 | 维护原则 |
| --- | --- | --- |
| `docs/project/` | 项目现状、技术栈、部署方式、运行边界 | 系统架构或部署方式变化时更新 |
| `docs/business/` | 领星下载、Excel 整理、命名、样式、异常处理等业务规则 | 每次业务规则变化必须同步更新 |
| `docs/harness/` | Agent Harness 工作流、复盘模板、规则沉淀、测试验收 | 每次线上问题复盘或流程升级时更新 |

## 当前核心文档

- `docs/project/项目交付说明.md`：当前系统已经实现了什么、怎么运行、技术栈是什么。
- `docs/business/领星自动化当前规则说明.md`：领星 RPA 和 Excel 处理的业务规则。
- `docs/harness/README.md`：RPA Agent Harness 总方案。
- `docs/harness/自动化需求交付规范.md`：后续自动化需求应该如何提供原始文件、基础数据和目标结果。
- `docs/harness/工作流说明.md`：从需求到上线的固定闭环。
- `docs/harness/失败复盘模板.md`：每次失败如何记录、修复和沉淀。
- `docs/harness/规则沉淀规范.md`：哪些经验要写入文档、测试或检查器。
- `docs/harness/测试与验收规范.md`：每次改动应该怎么验证。
- `docs/harness/审查与提交规范.md`：提交前检查、PR 模板和审查重点。

## 文档清理结论

本次整理没有把所有内容合并成一份大文档，原因是三类文档职责不同：

- 项目交付说明用于了解系统现状。
- 业务规则说明用于约束领星和 Excel 处理逻辑。
- Harness 文档用于约束以后怎么提交需求、执行任务、复盘失败和沉淀资产。

旧的根目录文档已迁入 `docs/` 对应目录，根目录不再保留重复副本，避免以后出现“改了旧文件但系统按新文件走”的混乱。
